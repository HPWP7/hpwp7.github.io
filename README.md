# ITEna Solutions Website

A modern, responsive corporate website for ITEna Solutions, redesigned with a polished visual identity and structured content for GitHub Pages.

## Overview

The website now presents ITEna Solutions as a professional software company with:
- a strong home page hero section
- a dedicated Solutions page
- a Contact Us page with address, email, phone, and map
- a clean, modern visual style with a dark navy base and red accent color
- responsive layouts for desktop, tablet, and mobile devices

## Current Pages

### Home
The main landing page highlights:
- the company story and mission
- a modern hero section with the tagline “We Make Technology Work For You”
- key statistics
- an overview of the company’s focus areas
- a strong call-to-action section

### Solutions
The solutions page provides a more detailed overview of:
- insurance-focused software
- online payments and payment integrations
- enterprise software and future SaaS offerings

### Contact Us
The contact page includes:
- business contact details
- email and phone information
- office location in Nicosia, Cyprus
- an embedded map

## Structure

The site is organized as a simple static website with these main files:

- index.html — homepage
- Solutions.html — solutions overview
- contactUs.html — contact information page
- CNAME — custom domain configuration
- assets/ — logo and favicon
- robots.txt / sitemap.xml — SEO configuration
- README.md — project documentation

## Design Notes

The new redesign uses:
- a contemporary layout with strong typography
- clear section spacing and hierarchy
- a refined dark/light contrast system
- red accent highlights for calls to action and key emphasis
- a professional corporate tone suited to enterprise clients

## Technical Details

- Static HTML website
- Responsive layout
- No framework dependency required
- Ready for GitHub Pages deployment

## Deployment

This repository is intended to be published directly through GitHub Pages from the repository root.

1. Push the latest changes to GitHub
2. Open the repository settings
3. Enable GitHub Pages if it is not already active
4. Publish from the main branch/root folder
